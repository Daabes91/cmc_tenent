## Analytics Verification

### Browser DevTools
1. Open DevTools → Network tab
2. Filter for analytics requests
3. Click each CTA button
4. Verify event is sent with:
   - Event name: `cta_click`
   - Category: Section name
   - Label: Button text
   - Value: Destination URL

### Expected Events
- `hero_section` - Primary/Secondary CTAs
- `features_section_cta` - Start Trial/Schedule Demo
- `testimonials_section_cta` - Get Started/Talk to Sales
- `pricing_section_cta` - Contact Sales/View Docs
- `integrations_section` - View All Integrations
- `final_cta_section` - Book Strategy Call

## Common Issues & Solutions

### Issue: CTA not appearing
**Solution:** Check that component is imported and variant prop is correct

### Issue: Wrong button text
**Solution:** Verify language context and check copy object

### Issue: Button not clickable
**Solution:** Check z-index and pointer-events CSS

### Issue: Analytics not tracking
**Solution:** Verify GA4 is configured and event handler is called

### Issue: Email link not working
**Solution:** Check SALES_EMAIL constant and mailto format

### Issue: RTL not working
**Solution:** Verify language context and RTL styles

## Quick Test Script

Run this quick test on each CTA section:

1. **Visual Check**
   - [ ] Section is visible
   - [ ] Text is healthcare-specific
   - [ ] Buttons are styled correctly
   - [ ] Layout is responsive

2. **Interaction Check**
   - [ ] Primary button navigates correctly
   - [ ] Secondary button navigates correctly
   - [ ] Hover states work
   - [ ] Focus states work

3. **Analytics Check**
   - [ ] Click sends analytics event
   - [ ] Event properties are correct

4. **Language Check**
   - [ ] Switch to Arabic
   - [ ] Text is translated
   - [ ] RTL layout works
   - [ ] Buttons still function
